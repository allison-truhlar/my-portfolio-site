Instructor Dashboard

My instructor dashboard app allows the user to connect to a course they are currently teaching on the Canvas Learning Management System via the Canvas API and view key information.

Project Overview

This app was inspired by my experience as an instructor for a course enrolling between 300 and 400 students each semester. I remembered facing challenges in efficiently identifying and reaching out to students needing extra support in the course. The default features of the learning management system used by the university at the time were insufficient for this task.

Problem Statement: As a course instructor, I need to be able to identify any students needing extra support in the course and contact them easily.

Product Solution: A dashboard that shows me key student metrics at a glance and allows me to email students in need of support.

Use Cases:
1. Visualize course progress based on today’s date and due dates for course assignments.
2. Identify students who are missing assignments.
3. Identify students with a low average.
4. Open an email to a student I need to check in with.

Challenges and Lessons Learned

Refactoring components for reusability: I originally built the app with two separate components for student progress and student grades. They ended up looking…nearly identical…so I knew this was an opportunity to make better use of React component reusability. I refactored the code to pull out specific logic related to finding students with missing assignments vs. low grades into functions that then could be passed down as props to two instances of one StudentDataModule component.

Learning NodeJS and Express: This was my first foray into full stack development, although this was not the intention from the start of the project! The Canvas API uses Cross-Origin Resource Sharing (CORS), a security feature allowing a server to specify which domains are allowed to access the server’s data. This meant I could not simply ask the Canvas API for data from a front-end app. Previously, I had only worked with APIs without CORS enabled. To overcome this, I found an open-source repository on Github that set up a server in NodeJS to access specified Canvas API endpoints and then serve them to my front-end. Using this code and adapting it to my specific use cases had a steep learning curve, but thanks to the helpful tutorial in the repository and the Canvas API community forums, I was able to get my server working with the Canvas API and my front-end.

Since creating this project, I am determined to expand my skills in NodeJS and Express and to build more extensive full stack apps in the future.
