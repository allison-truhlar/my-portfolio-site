Chingu Voyage Experience

Over 6 one-week sprints, I am collaborated in a team of 6 front end developers to develop an app visualizing NASA meteorite strike data.

Experience Overview

Chingu is a volunteer-run organization that provides free collaborative learning experiences to aspiring software developers. During August and September 2023, I worked in a team of 6 front end developers and to build and deploy our MVP over the course of 6 one-week sprints, or a "voyage".

We were asked to create an app to search and visualize meteorite strike data. The app specifications required three components: a component to search the data, a component to display the search results in a data table, and a component to display some summary metrics for the search results.

I was responsible for creating the a data table component with one other teammate. My major contribution was identifying an efficient method to reverse geocode latitude and longitude data for each meteorite strike. With my background in environmental engineering, I have experience working with geospatial data in R and ArcGIS - I was excited to explore tools for this in JavaScript.

Challenges and Lessons Learned

Optimizing performance: I quickly discovered that reverse geocoding is a relatively time intensive task. This resulted in a poor user experience - specifically, the detailed data table populated several seconds after the rest of the app components. Utlimately, I was able to solve this issue by using a different API for reverse geocoding.

The alternate reverse geocoding API I tried had a higher rate-limit for requests which resulted in a barely noticeable loading time for the locations. However, the tradeoff was that the location provided in the response was less granular - for example, you might only receive state-level information as opposed to city-level. After discussion with my team, we decided this tradeoff was acceptable. We prioritized user experience of viewing the data quickly because the app is purely for informational purposes - the user would not rely on the location data for navigation or another similar purpose.

Using GIT: This was my first experience working in a team to create an app and, therefore, was my first experience with practical use of GIT for version management.
