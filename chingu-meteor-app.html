Chingu Voyage Experience

Over 6 one-week sprints, I am collaborating in a team of 6 front end developers to develop an app to visualize NASA meteorite strike data.

Experience Overview

Chingu is a volunteer-run organization that provides free collaborative learning experiences to aspiring software developers. I was placed in a team of 6 front end developers and challenged to build and deploy our MVP over the course of 6 one-week sprints.

I was particularly excited about the topic of our app - accessing, searching, and displaying meteorite strike data. With my background in environmental engineering, I have experience working with geospatial data in R and ArcGIS - I was excited to explore tools for this in JavaScript.

I was responsible for a data table display component with one other teammate. The table displays one row of data for each meteorite strike returned by the user's search requirements, handled in another component. My personal focus was identifying an efficient method to reverse geocode latitude and longitude data for each meteorite strike.

Challenges and Lessons Learned

Optimizing performance: I quickly discovered that reverse geocoding is a relatively time intensive task. This resulted in a poor user experience - specifically, the detailed data table populated several seconds after the rest of the app components. Utlimately, I was able to solve this issue by using a different API for reverse geocoding.

The alternate reverse geocoding API I tried had a higher rate-limit for requests which resulted in a barely noticeable loading time for the locations. However, the tradeoff was that the location provided in the response was less granual - for example, you might only receive state-level information as opposed to city-level. After discussion with my team, we decided this tradeoff was acceptable. We prioritized user experience of viewing the data quickly because the app is purely for informational purposes - the user would not rely on the location data for navigation or another similar purpose.

Using GIT: This was my first experience working in a team to create an app, and therefore was my first experience with practical use of GIT for version management.
